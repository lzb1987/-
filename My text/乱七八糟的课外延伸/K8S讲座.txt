Kubernetes(k8s)

一、Docker——容器技术  最火的服务端技术
  2013年发布，开源
  1.什么是Docker？
  · 容器和镜像
    容器技术的核心功能，就是通过约束和修改进程的动态表实现，从而为其创造出一个“边界”。
    Cgroups 技术 —— 是用来制造约束的主要手段，分配系统资源
      - 限制内存使用上限，（），文件系统的缓存，磁盘分组，审计和统计，挂载和恢复进程
    Namespace技术 —— 修改进程视图的主要方法
      docker run —— 跑镜像
      一个进程，在上面跑的进程合并为进程组
      - 通过命名空间隔离一系列资源
      
    容器实际上是一个有LinuxNamespace,LinuxCgroup和rootft三种技术构建出来的进程的隔离环境
    
  2.为什么选择它
    ·更高效的虚拟化       空间利用率
    ·更轻松的迁移和拓展    pull+run
    ·更快速的部署和交付
    ·简单的管理
    
    perl go c java python
    
    普罗米修斯监控组件
    
  3.怎么用
  
  K8S从容器到容器云  容器本身没有价值，有价值的师“容器编排”
  容器编排、调度、容器云等
  POD K8S最小调度单位
  RC 是K8S系统中的核心概念之一
   - 定义了POD期待的副本数
   - 用于筛选POD的标签
   - 
  
  Replica Set 支持基于集和的标签选择器
  Service是最核心的资源对象之一
  
  Kubeadm是K8S官方提供的快速安装和初始化k8s集群的工具
